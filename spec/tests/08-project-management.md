# UC-08: プロジェクト管理

## 概要

プロジェクトファイルの保存・読み込み・別名保存、および未保存変更の確認ダイアログを検証する。

---

## ユースケース一覧

### UC-08-01: 文字起こし前は保存ボタンが無効

**前提条件**: アプリが起動している（文字起こし未実行）

**期待結果**:
- 左ペインの「💾 プロジェクトを保存」ボタンが **無効**
- メニュー「ファイル > プロジェクトを保存」が **無効**
- メニュー「ファイル > プロジェクトを別名で保存」が **無効**

### UC-08-02: 文字起こし後に保存ボタンが有効化

**前提条件**: モックデータが注入済み（文字起こし後の状態）

**期待結果**:
- 左ペインの「💾 プロジェクトを保存」ボタンが **有効**
- メニュー「ファイル > プロジェクトを保存」が **有効**
- メニュー「ファイル > プロジェクトを別名で保存」が **有効**

### UC-08-03: プロジェクトを別名で保存

**前提条件**:
- モックデータが注入済み
- `pywebview.api.save_project_as` をモックしてパスを返す設定済み

**操作**: 「💾 プロジェクトを保存」ボタンをクリックする（初回）

**期待結果**:
- `pywebview.api.save_project_as()` が呼び出される
- 保存成功後に dirty 状態がクリアされる
- ウィンドウタイトルから `*` が消える

### UC-08-04: プロジェクトを上書き保存 (Ctrl+S)

**前提条件**:
- プロジェクトが既にファイルに保存済みの状態（`_current_project_path` が設定済み）
- セグメントを編集して dirty 状態

**操作**: `Ctrl+S` を押す

**期待結果**:
- `pywebview.api.save_project()` が呼び出される（ダイアログなし）
- dirty 状態がクリアされる

### UC-08-05: プロジェクトを開く

**前提条件**:
- `pywebview.api.load_project` をモックして有効なプロジェクトデータを返す設定済み

**操作**: 「📂 プロジェクトを開く」ボタンまたはメニューをクリックする

**期待結果**:
- セグメント一覧にロードされたデータが表示される
- 「💾 exoファイルに保存」ボタンが有効になる
- dirty 状態がクリアされている

### UC-08-06: 未保存変更がある状態でプロジェクトを開こうとする

**前提条件**:
- モックデータが注入済みでセグメントが編集された (dirty 状態)

**操作**: 「📂 プロジェクトを開く」ボタンをクリックする

**期待結果**:
- 未保存確認ダイアログ (`#save-confirm-modal`) が表示される
- 「はい」「いいえ」「キャンセル」の3ボタンが表示される

### UC-08-07: 未保存確認ダイアログ: 「はい」（保存して続行）

**前提条件**: 未保存確認ダイアログが表示されている

**操作**: 「はい」ボタンをクリックする

**期待結果**:
- プロジェクト保存処理が実行される
- その後に新しいプロジェクト読み込みが実行される

### UC-08-08: 未保存確認ダイアログ: 「いいえ」（保存せず続行）

**前提条件**: 未保存確認ダイアログが表示されている

**操作**: 「いいえ」ボタンをクリックする

**期待結果**:
- 保存せずに新しいプロジェクト読み込みが実行される

### UC-08-09: 未保存確認ダイアログ: 「キャンセル」

**前提条件**: 未保存確認ダイアログが表示されている

**操作**: 「キャンセル」ボタンをクリックする

**期待結果**:
- ダイアログが閉じる
- 元の状態（dirty のまま）を維持する

### UC-08-10: アプリ終了時の未保存確認

**前提条件**: dirty 状態でアプリを閉じようとする

**操作**: ウィンドウの閉じるボタンを押す

**期待結果**:
- 未保存確認ダイアログが表示される

### UC-08-11: ウィンドウタイトルの更新

**前提条件**: プロジェクトが「myproject.json」として保存済み

**期待結果**:
- タイトルバーに `aviutl-whisper - myproject` と表示される

**前提条件**: セグメントを編集して dirty 状態

**期待結果**:
- タイトルバーに `*aviutl-whisper - myproject` と表示される

## テスト実装メモ

```python
def test_save_buttons_disabled_before_transcription(page):
    assert page.locator("#btn-save-project").is_disabled()

def test_unsaved_dialog_appears(page, inject_mock_segments):
    # セグメントを編集して dirty にする
    page.click("#segment-table-body tr:first-child")
    page.fill("#seg-edit-text", "変更")
    page.click("#btn-seg-apply")
    # プロジェクトを開こうとする
    page.click("#btn-load-project")
    assert page.locator("#save-confirm-modal").is_visible()
```
