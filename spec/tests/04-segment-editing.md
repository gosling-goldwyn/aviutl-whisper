# UC-04: セグメント編集

## 概要

文字起こし結果のセグメント（話者・時刻・テキスト）の編集操作を確認する。

---

## 前提: モックデータ注入

文字起こし後の状態を模倣するため、`page.evaluate()` で以下を実行してからテストを開始する:

```js
updateSegments([
  { start: 0.0,  end: 3.5,  speaker: "Speaker 1", text: "こんにちは、テストです。" },
  { start: 3.8,  end: 7.2,  speaker: "Speaker 2", text: "はじめまして。よろしくお願いします。" },
  { start: 7.5,  end: 12.0, speaker: "Speaker 1", text: "今日はよい天気ですね。" }
]);
```

---

## ユースケース一覧

### UC-04-01: セグメント一覧の表示

**前提条件**: モックデータが注入済み

**期待結果**:
- テーブルに3行のセグメントが表示される
- 各行に「時刻」「話者」「テキスト」が表示される
- 空メッセージ (`#segment-table-empty`) が非表示

### UC-04-02: セグメントのクリックによる編集パネル表示

**前提条件**: モックデータが注入済み

**操作**: テーブルの1行目をクリックする

**期待結果**:
- セグメント編集パネル (`#seg-editor`) が表示される
- 「話者」セレクトに「Speaker 1」が選択されている
- 「開始」入力に `0` が表示される
- 「終了」入力に `3.5` が表示される
- 「テキスト」テキストエリアに「こんにちは、テストです。」が表示される

### UC-04-03: テキストの編集と適用

**前提条件**: セグメント編集パネルが表示済み

**操作**:
1. テキストエリアの内容を「修正後のテキスト。」に変更する
2. 「✔ 適用」ボタンをクリックする

**期待結果**:
- セグメント一覧の対象行のテキストが「修正後のテキスト。」に更新される
- dirty 状態になる（ウィンドウタイトルに `*` が追加される）

### UC-04-04: テキストのリアルタイム反映 (デバウンス)

**前提条件**: セグメント編集パネルが表示済み

**操作**: テキストエリアに文字を入力する（300ms 待つ）

**期待結果**:
- 自動でプレビューが更新される（「✔ 適用」ボタンを押さなくても）

### UC-04-05: 話者の変更

**前提条件**: セグメント編集パネルが表示済み

**操作**:
1. 「話者」セレクトを「Speaker 2」に変更する
2. 「✔ 適用」ボタンをクリックする

**期待結果**:
- セグメント一覧の対象行の話者が「Speaker 2」に更新される

### UC-04-06: 時刻の変更

**前提条件**: セグメント編集パネルが表示済み

**操作**:
1. 「開始」を `1.0` に変更し「終了」を `4.0` に変更する
2. 「✔ 適用」ボタンをクリックする

**期待結果**:
- セグメント一覧の対象行の時刻が更新される

### UC-04-07: セグメントの追加

**前提条件**: セグメント編集パネルが表示済み

**操作**: 「＋ 追加」ボタンをクリックする

**期待結果**:
- セグメント数が1増える
- 新しいセグメントが一覧に追加される

### UC-04-08: セグメントの削除

**前提条件**: セグメント編集パネルが表示済み

**操作**: 「🗑 削除」ボタンをクリックする

**期待結果**:
- 対象セグメントが一覧から削除される
- セグメント数が1減る

### UC-04-09: 前のセグメントと結合

**前提条件**: 2番目以降のセグメントが選択済み、かつ前のセグメントと話者が同じ

**操作**: 「◀ 前と結合」ボタンをクリックする

**期待結果**:
- 2つのセグメントが1つに結合される
- テキストが連結される
- 開始時刻・終了時刻が正しく統合される

### UC-04-10: 後のセグメントと結合

**前提条件**: 最後以外のセグメントが選択済み、かつ後のセグメントと話者が同じ

**操作**: 「後と結合 ▶」ボタンをクリックする

**期待結果**:
- 2つのセグメントが1つに結合される

### UC-04-11: 異なる話者では結合ボタンが無効

**前提条件**: セグメント2（Speaker 2）が選択済み、隣接セグメントが Speaker 1

**期待結果**:
- 「◀ 前と結合」ボタンが **無効** (disabled)
- 「後と結合 ▶」ボタンが **無効** (disabled)

### UC-04-12: セグメント音声再生

**前提条件**: モックデータが注入済み、`pywebview.api.play_segment_audio` がモック済み

**操作**: 「🔊 再生」ボタンをクリックする

**期待結果**:
- `pywebview.api.play_segment_audio(index)` が呼び出される

## テスト実装メモ

```python
def test_segment_edit_text(page, inject_mock_segments):
    page.click("#segment-table-body tr:first-child")
    page.fill("#seg-edit-text", "修正後のテキスト。")
    page.click("#btn-seg-apply")
    row_text = page.locator("#segment-table-body tr:first-child td.col-text").inner_text()
    assert row_text == "修正後のテキスト。"
```
