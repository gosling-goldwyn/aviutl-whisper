# UC-10: Undo / Redo

## 概要

セグメント編集操作の Undo (取り消し) と Redo (やり直し) を確認する。

---

## 前提

- Undo スタックの最大深度は 50
- 対象操作: テキスト編集、話者変更、時刻変更、セグメント追加・削除・結合
- テキストボックス内のネイティブ Ctrl+Z はブラウザデフォルトを使用（アプリの Undo は発火しない）

---

## ユースケース一覧

### UC-10-01: 初期状態では Undo/Redo が無効

**前提条件**: アプリが起動している

**期待結果**:
- メニュー「編集 > 元に戻す」が **無効**
- メニュー「編集 > やり直し」が **無効**

### UC-10-02: 編集後に Undo が有効化

**前提条件**: モックデータが注入済み

**操作**: セグメントのテキストを変更して「✔ 適用」をクリックする

**期待結果**:
- メニュー「編集 > 元に戻す」が **有効**

### UC-10-03: テキスト編集の Undo

**前提条件**: モックデータが注入済み

**操作**:
1. セグメント 1 のテキストを「変更後」に編集して適用する
2. Undo を実行する（`Ctrl+Z` またはメニュー）

**期待結果**:
- セグメント 1 のテキストが元の「こんにちは、テストです。」に戻る

### UC-10-04: Undo 後に Redo が有効化

**前提条件**: Undo を実行した後

**期待結果**:
- メニュー「編集 > やり直し」が **有効**

### UC-10-05: Redo の実行

**前提条件**: Undo を実行した後

**操作**: Redo を実行する（`Ctrl+Y` またはメニュー）

**期待結果**:
- テキストが再度「変更後」に戻る
- Redo が再び **無効** になる

### UC-10-06: 新しい操作後に Redo スタックがクリア

**前提条件**: Undo 後の状態（Redo 可能）

**操作**: 別のセグメントを編集する

**期待結果**:
- Redo が **無効** になる（新しい操作でリセット）

### UC-10-07: セグメント追加の Undo

**前提条件**: モックデータが注入済み

**操作**:
1. 「＋ 追加」ボタンでセグメントを追加する
2. Undo を実行する

**期待結果**:
- セグメント数が元に戻る（追加前の数）

### UC-10-08: セグメント削除の Undo

**前提条件**: モックデータが注入済み

**操作**:
1. セグメントを削除する
2. Undo を実行する

**期待結果**:
- セグメントが復元される

### UC-10-09: EXO 設定変更の Undo

**前提条件**: モックデータが注入済み

**操作**:
1. フォントサイズを変更する
2. Undo を実行する

**期待結果**:
- フォントサイズが元の値に戻る

### UC-10-10: テキストエリア内では Ctrl+Z がネイティブ動作

**前提条件**: セグメント編集パネルのテキストエリアにフォーカスがある

**操作**: テキストを入力してから `Ctrl+Z` を押す

**期待結果**:
- テキストエリア内の文字が1操作分取り消される（ブラウザのネイティブ Undo）
- アプリのセグメント Undo は発火しない

## テスト実装メモ

```python
def test_undo_redo_menu_initial_state(page):
    page.click("#menu-edit-entry .menu-entry-btn")
    assert page.locator("#menu-undo").is_disabled()
    assert page.locator("#menu-redo").is_disabled()

def test_undo_text_edit(page, inject_mock_segments):
    page.click("#segment-table-body tr:first-child")
    page.fill("#seg-edit-text", "変更後")
    page.click("#btn-seg-apply")
    # Undo
    page.keyboard.press("Control+z")
    text = page.locator("#segment-table-body tr:first-child .col-text").inner_text()
    assert text == "こんにちは、テストです。"
```
