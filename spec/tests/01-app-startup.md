# UC-01: アプリ起動・初期 UI

## 概要

アプリケーションが正常に起動し、初期状態の UI が正しく表示されることを確認する。

---

## ユースケース一覧

### UC-01-01: アプリ起動時の初期表示

**前提条件**: アプリが起動している

**操作**: なし（起動直後の状態）

**期待結果**:
- タイトルバーに `aviutl-whisper` と表示される
- ヘッダーに `🎙️ aviutl-whisper` と表示される
- 「ファイルを選択」ボタンが表示される
- ファイル名ラベルに「未選択」と表示される
- 「▶ 開始」ボタンが **無効** (disabled) 状態
- 「💾 exoファイルに保存」ボタンが **無効** 状態
- セグメント一覧は空で「文字起こしを実行するとセグメントが表示されます」というメッセージが表示される
- プレビューエリアに「文字起こしを実行するとプレビューが表示されます」が表示される
- 進捗バーエリアが非表示

### UC-01-02: デバイス情報の表示

**前提条件**: アプリが起動し pywebview の API が準備完了している

**操作**: 画面下部のデバイス情報エリアを確認する

**期待結果**:
- デバイス情報 (「CPU」または「GPU (CUDA)」) が表示される

### UC-01-03: フォント選択肢の読み込み

**前提条件**: アプリが起動し pywebview の API が準備完了している

**操作**: EXO詳細設定セクションの「フォント」セレクトボックスを確認する

**期待結果**:
- フォントドロップダウンに1件以上の選択肢が表示される
- 「MS UI Gothic」または同等のシステムフォントが含まれる

### UC-01-04: メニューバーの表示

**前提条件**: アプリが起動している

**操作**: メニューバーを確認する

**期待結果**:
- 「ファイル(F)」メニューボタンが表示される
- 「編集(E)」メニューボタンが表示される

### UC-01-05: 話者カラー初期表示

**前提条件**: アプリが起動している

**操作**: 「🎨 話者ごとの色設定」セクションを確認する

**期待結果**:
- 話者数のデフォルト (自動推定 = 2人相当) に応じた色設定行が表示される
- 各行に文字色・縁色のカラーピッカーが表示される

## テスト実装メモ

```python
def test_initial_ui(page):
    # ボタンの有効/無効状態
    assert page.locator("#btn-start").is_disabled()
    assert page.locator("#btn-save").is_disabled()
    # ファイル名ラベル
    assert page.locator("#file-name").inner_text() == "未選択"
    # セグメント空メッセージ
    assert page.locator("#segment-table-empty").is_visible()
    # プレビュープレースホルダー
    assert page.locator("#preview-placeholder").is_visible()
```
