# UC-06: シーンプレビュー

## 概要

文字起こし結果の各セグメントのシーンプレビュー画像表示とナビゲーションを確認する。

---

## 前提: モックデータ注入

セグメントデータが注入され、プレビュー生成 API がモック済みであること。

---

## ユースケース一覧

### UC-06-01: 初期状態のプレビューエリア

**前提条件**: アプリが起動している (文字起こし前)

**期待結果**:
- プレビュープレースホルダー (`#preview-placeholder`) が表示されている
- プレビュー画像 (`#preview-image`) が非表示
- 「◀」「▶」ナビゲーションボタンが **無効**
- セグメント情報 (`#preview-seg-info`) が `- / -` と表示される

### UC-06-02: 文字起こし後のプレビュー表示

**前提条件**: モックセグメントデータが注入され、プレビュー API がデータ URI を返すようにモック済み

**操作**: 最初のセグメントが選択された状態

**期待結果**:
- プレビュー画像 (`#preview-image`) が表示される
- プレースホルダーが非表示
- セグメント情報に `1 / 3` のような表示がある
- 「◀」ボタンが **無効** (最初のセグメントでは前がない)
- 「▶」ボタンが **有効**

### UC-06-03: 次のセグメントへのナビゲーション

**前提条件**: 最初のセグメントが表示されている

**操作**: 「▶」ボタンをクリックする

**期待結果**:
- セグメント情報が `2 / 3` に変わる
- プレビュー画像が更新される
- 「◀」ボタンが **有効** になる

### UC-06-04: 前のセグメントへのナビゲーション

**前提条件**: 2番目以降のセグメントが表示されている

**操作**: 「◀」ボタンをクリックする

**期待結果**:
- セグメント情報が前の番号に変わる
- プレビュー画像が更新される

### UC-06-05: 最後のセグメントでの「▶」ボタン無効化

**前提条件**: 最後のセグメントが表示されている

**期待結果**:
- 「▶」ボタンが **無効**

### UC-06-06: セグメント一覧クリックによるプレビュー同期

**前提条件**: モックデータが注入済み

**操作**: セグメント一覧の3行目をクリックする

**期待結果**:
- セグメント情報が `3 / 3` に変わる
- 3番目のセグメントのプレビューが表示される
- セグメント編集パネルに3番目のデータが反映される

### UC-06-07: 背景画像なしのプレビュー

**前提条件**: 背景画像が未設定

**期待結果**:
- プレビューが黒背景または透明背景で字幕のみ表示される

### UC-06-08: 背景画像ありのプレビュー

**前提条件**:
- 背景画像ファイルが選択済み (`pywebview.api.select_image_file` をモック)

**期待結果**:
- プレビューに背景画像が合成される

def test_preview_navigation(page, inject_mock_segments):
    # 初期状態
    assert page.locator("#btn-next-seg").is_disabled() == False
    assert page.locator("#btn-prev-seg").is_disabled()

    # 次へ
    page.click("#btn-next-seg")
    assert page.locator("#preview-seg-info").inner_text() == "2 / 3"
    assert not page.locator("#btn-prev-seg").is_disabled()
